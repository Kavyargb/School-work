1. Understanding Functions

Topic: Defining and calling custom functions to create modular and reusable code.
What is a Function?

A function is a named, reusable block of code that performs a specific, well-defined task. The core principle behind functions is DRY (Don't Repeat Yourself). By encapsulating logic into a function, you can call it multiple times from different parts of your program without rewriting the code. This makes your programs shorter, more readable, and easier to maintain.

Defining and Calling a Function

You define a function using the def keyword, followed by a function name, parentheses (), and a colon :. The code block within the function must be indented.

# Defining the function
def greet():
    print("Hello from a function!")
    print("Welcome to the module.")

# Calling the function
greet()

When `greet()` is called, the Python interpreter jumps to the function's definition and executes the code inside it. After the function finishes, the program execution returns to the line where the function was called.

2. Parameters & Arguments

Topic: Passing data into functions using parameters and arguments.

Functions become truly powerful when you can pass data into them. This is done using parameters and arguments.

  • Parameters: The variables listed inside the function's parentheses in its definition. They are placeholders for the data the function expects to receive.
  • Arguments: The actual values that are passed to the function when it is called.
# 'name' is a parameter
def greet_user(name):
    print(f"Hello, {name}!")

# "Alice" is a positional argument
greet_user("Alice")

# 'name="Bob"' is a keyword argument
greet_user(name="Bob")
Default Parameter Values

You can provide a default value for a parameter, making it optional when the function is called. If an argument for that parameter is not provided, the default value is used.

def describe_pet(pet_name, animal_type='dog'):
    print(f"I have a {animal_type} named {pet_name}.")

describe_pet("Fido")                    # Uses the default animal_type
describe_pet("Whiskers", "cat")     # Overrides the default

3. The `return` Statement

Topic: Getting data out of a function using the `return` statement.

While `print()` displays information to the console, the `return` statement sends a value back from the function to the place where it was called. This returned value can be stored in a variable or used in other expressions, making functions a fundamental building block for complex programs.

Figure: Arguments flow in, the function processes them, and a value flows out via `return`.

def square(number):
    return number * number

result = square(4)  # The value 16 is returned and stored in 'result'
print(result)       # Output: 16
print(square(5)) # The returned value 25 is passed directly to print()

A function can only have one `return` statement that executes. Once a `return` is hit, the function exits immediately. It can, however, return any kind of Python object: a number, a string, a list, a dictionary, or even another function.

4. Scope and Docstrings

Topic: Understanding variable scope and documenting functions with docstrings.
Variable Scope

Scope refers to the region of a program where a variable is accessible. Variables defined inside a function have local scope, meaning they only exist and can only be accessed from within that function. Variables defined outside of any function have global scope.

global_var = "I'm outside!"

def my_function():
    local_var = "I'm inside!"
    print(global_var) # Can access global variables
    print(local_var)

my_function()
# print(local_var) # This would cause a NameError
Docstrings

A docstring is a string literal that occurs as the first statement in a module, function, class, or method definition. It's a crucial part of good coding practice, as it provides documentation on what the function does. It is enclosed in triple quotes """...""".

def calculate_area(length, width):
    """Calculate the area of a rectangle.

    Args:
        length (int or float): The length of the rectangle.
        width (int or float): The width of the rectangle.

    Returns:
        int or float: The calculated area of the rectangle.
    """
    return length * width

You can access a function's docstring using the `help()` function or the `__doc__` attribute: `help(calculate_area)` or `print(calculate_area.__doc__)`.

Q1. Write a function `is_palindrome(s)` that takes a string `s` and returns `True` if the string is a palindrome and `False` otherwise. A palindrome is a word that reads the same forwards and backward (e.g., "racecar").
Solution

This can be solved efficiently by comparing the string to its reverse, which can be obtained using slicing.

def is_palindrome(s):
    """Checks if a string is a palindrome."""
    # Clean the string: remove spaces and convert to lowercase for a robust check
    cleaned_s = ''.join(char for char in s if char.isalnum()).lower()
    return cleaned_s == cleaned_s[::-1]

print(is_palindrome("racecar"))  # True
print(is_palindrome("hello"))    # False
print(is_palindrome("A man a plan a canal Panama")) # True
Q2. Write a function `calculate_factorial(n)` that computes the factorial of a non-negative integer `n`. The factorial is the product of all positive integers up to `n` (e.g., 5! = 5 * 4 * 3 * 2 * 1 = 120). The factorial of 0 is 1.
Solution

This requires a loop and an "accumulator" variable that keeps track of the running product.

def calculate_factorial(n):
    """Computes the factorial of a non-negative integer."""
    if n < 0:
        return "Factorial is not defined for negative numbers"
    if n == 0:
        return 1
    
    factorial = 1
    for i in range(1, n + 1):
        factorial *= i
    return factorial

print(calculate_factorial(5)) # 120
print(calculate_factorial(0)) # 1
Q3. Write a function `is_prime(num)` that checks if a number is prime. A prime number is a number greater than 1 that has no positive divisors other than 1 and itself. The function should return `True` if it's prime, and `False` otherwise.
Solution

The logic involves checking for divisibility from 2 up to the square root of the number. If any number in that range divides it evenly, it's not prime.

import math

def is_prime(num):
    """Checks if a number is prime."""
    if num <= 1:
        return False
    if num == 2:
        return True
    if num % 2 == 0:
        return False # All other even numbers are not prime

    # Check for odd divisors from 3 up to the square root of num
    for i in range(3, int(math.sqrt(num)) + 1, 2):
        if num % i == 0:
            return False
    
    return True

print(is_prime(29)) # True
print(is_prime(30)) # False
Q4. Write a function `find_max(numbers)` that takes a list of numbers and returns the largest number in the list *without* using the built-in `max()` function. The function should return `None` if the list is empty.
Solution

This tests algorithmic thinking. We initialize a variable with the first element, then loop through the rest, updating our variable whenever we find a larger number.

def find_max(numbers):
    """Finds the maximum number in a list without using max()."""
    if not numbers: # Check if the list is empty
        return None
    
    max_so_far = numbers[0]
    for num in numbers[1:]:
        if num > max_so_far:
            max_so_far = num
            
    return max_so_far

print(find_max([1, 5, 2, 9, 3])) # 9
print(find_max([])) # None
Q5. Write a function `count_frequency(items)` that takes a list and returns a dictionary where keys are the unique items in the list and values are their frequencies (how many times they appear). For example, `count_frequency(['a', 'b', 'a', 'c', 'b', 'a'])` should return `{'a': 3, 'b': 2, 'c': 1}`.
Solution

This is a practical problem that combines loops and dictionaries. The `dictionary.get(key, default)` method is perfect for this task.

def count_frequency(items):
    """Counts the frequency of items in a list."""
    frequency_dict = {}
    for item in items:
        # Get the current count (or 0 if it's not there), add 1, and update
        frequency_dict[item] = frequency_dict.get(item, 0) + 1
    return frequency_dict

print(count_frequency(['a', 'b', 'a', 'c', 'b', 'a']))
Q6. Write a function `sum_all(*args)` that can accept any number of numeric arguments and return their sum. For example, `sum_all(1, 2, 3)` should return `6`, and `sum_all(10, 20)` should return `30`.
Solution

The `*args` syntax allows a function to accept a variable number of positional arguments. These arguments are collected into a tuple.

def sum_all(*args):
    """Accepts any number of numeric arguments and returns their sum."""
    # 'args' will be a tuple containing all passed arguments, e.g., (1, 2, 3)
    return sum(args)

print(sum_all(1, 2, 3, 4, 5)) # 15
print(sum_all(10, 20))        # 30
Q7. Write a function `create_user(username, role='guest', is_active=False)` that returns a dictionary representing a user. The `username` should be a required argument, while `role` and `is_active` should have default values.
Solution

This question tests the practical application of default arguments to create flexible functions.

def create_user(username, role='guest', is_active=False):
    """Creates a user dictionary with default roles and status."""
    return {
        'username': username,
        'role': role,
        'is_active': is_active
    }

# Call with only the required argument
print(create_user('john_doe'))
# Expected: {'username': 'john_doe', 'role': 'guest', 'is_active': False}

# Call overriding default arguments
print(create_user('admin_user', role='admin', is_active=True))
# Expected: {'username': 'admin_user', 'role': 'admin', 'is_active': True}
Q8. Write a function `fizzbuzz(n)` that prints numbers from 1 to `n`. For multiples of 3, print "Fizz" instead of the number. For multiples of 5, print "Buzz". For numbers which are multiples of both 3 and 5, print "FizzBuzz". This function does not need to return anything.
Solution

This classic problem tests conditional logic. The key is to check for the "FizzBuzz" case (multiples of both 3 and 5) first, otherwise it will never be reached.

def fizzbuzz(n):
    """Prints the FizzBuzz sequence up to n."""
    for i in range(1, n + 1):
        if i % 15 == 0: # Check for multiple of 3 and 5 first
            print("FizzBuzz")
        elif i % 3 == 0:
            print("Fizz")
        elif i % 5 == 0:
            print("Buzz")
        else:
            print(i)

fizzbuzz(20)
